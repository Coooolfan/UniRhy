# UniRhy 是什么

UniRhy（独一律）是一个**自托管的音乐流媒体平台**。它把一台你能掌控的服务器（家庭 NAS、VPS、内网主机皆可）变成你私人的音乐播放后端：管理资源、分发音频、在多设备间同步播放状态。

## 核心特性

- **完全自托管**：所有数据存储在你自己控制的服务器上，无任何外部服务依赖。
- **跨平台客户端**：Web、macOS、Windows、Android、iOS，统一播放体验。
- **多设备播放同步**：同一账号在多个设备上保持播放进度一致，离开/加入同步会话顺滑切换。
- **插件化扩展**：通过插件接入第三方元数据、转码、艺术家归一化等能力。
- **音乐资源管理**：扫描、转码、专辑/歌单/作品维度组织，支持本地与 OSS 存储节点。

## 适用场景

- 个人拥有较大体量音乐文件（数千至数万曲），希望集中管理而不是依赖云服务。
- 家庭/小团队希望共享音乐库，同时保留各自的播放进度与偏好。
- 对版权与隐私有要求，不希望使用境外或大型流媒体服务。
- 想在多端（手机、桌面、车机）保持播放进度连续。

## 不适用的场景

- 完全不愿运维的用户：UniRhy 需要一台可访问的服务器与 PostgreSQL。
- 仅依赖流媒体目录的用户：UniRhy 不提供版权曲库，需要你拥有合法的本地音频文件。
- 需要 DLNA / Cast / 公开分享链接等场景：当前版本聚焦私人同步播放，不包含这些功能。

## 当前状态

UniRhy 处于 **beta** 阶段。播放、同步、库管理等核心链路已可用并经过端到端测试覆盖；部分高级能力（AI 歌单、向量化推荐等）仍在路线图中。详见 [GitHub Releases](https://github.com/Coooolfan/UniRhy/releases)。
