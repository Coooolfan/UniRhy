UniRhy（独一律）是一个**自托管的音乐流媒体平台**。其支撑你在一台服务器（家庭 NAS、VPS、内网主机皆可）上部署你私人的音乐播放服务：管理资源、分发音频、在多设备间同步播放状态。

## 核心特性

- **完全自托管**：所有数据存储在你自己的服务器上，无任何外部服务依赖。
- **跨平台客户端**：Web、macOS、Windows、Android、iOS，全平台一致。
- **多设备播放同步**：同账号下多设备协同播放，播放进度误差齐平人耳感知。
- **插件化扩展**：统一、支持权限、文档完善的插件体系（WIP）。
- **音乐资源管理**：扫描、组织、转码，支持本地与对象存储。

## 整体结构

UniRhy 的运行与部署只需要两个服务，它们在文档提供的 `compose.yml` 中编排就绪，并直接面向 Docker 发行：

- **服务端**：Web 静态资源托管与后端服务，负责音乐库管理、播放分发与多端同步，管理本地音频文件。
- **数据库**：负责数据持久化。

服务端与客户端之间走标准的 HTTP 与 WebSocket，部署只要保证客户端能访问到服务端即可，家用 NAS、VPS、内网主机都行。

## 当前状态

UniRhy 处于 **beta** 阶段。播放、同步、库管理等核心链路已可用并经过端到端测试覆盖；部分高级能力仍在路线图中。详见 [GitHub Releases](https://github.com/Coooolfan/UniRhy/releases)。
