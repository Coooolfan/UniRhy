# 首次启动

server 启动并访问 `http://<服务器地址>:8654` 后，前端会检测到数据库尚未初始化，自动跳转到 `/init` 页面引导你完成两步初始化。

## Part 01：创建管理员账户

第一步需要创建唯一的管理员账户。这个账户拥有对实例的完全控制权——后续创建其他账户、配置存储节点、安装插件等都通过它进行。

| 字段   | 说明                                                   |
| ------ | ------------------------------------------------------ |
| 账号名 | 登录用户名，建议简短小写英文                           |
| 邮箱   | 用于找回密码或后续通知（当前版本仅作标识，无邮件功能） |
| 密码   | 至少 8 位，建议混合大小写、数字与符号                  |

> 当前版本不支持通过 UI 重置管理员密码，请务必妥善保存。

## Part 02：系统存储节点

第二步指定 UniRhy 用来存放扫描索引、缓存、转码产物的根路径。默认 `./data`，相对于 server 工作目录。

- Docker 部署：默认 server 工作目录在容器内为 `/app`，强烈建议改为映射进来的卷路径，例如 `/music`。需要先在 `docker-compose.yml` 里把宿主机目录挂载进容器，再在这里填入容器内路径。
- 裸机部署：填一个 server 用户可读写的绝对路径，例如 `/var/lib/unirhy/data`。

这个根路径会被作为 UniRhy 的**默认本地存储节点**。后续你可以在「设置 → 存储节点」里增加更多本地节点或 OSS（S3 兼容）节点，UniRhy 支持多存储节点并行扫描。

## 完成初始化

点击「初始化」后会自动跳转到登录页（`/login`），用刚刚创建的管理员账户登录即可进入主界面。

## 接下来

进入主界面后建议按以下顺序配置：

1. **存储节点**：「设置 → 存储节点」检查/补充节点配置。
2. **导入音乐**：把音乐文件放进存储节点的根路径下任意位置。文件结构没有强制要求，UniRhy 按音频内嵌元数据组织。
3. **触发扫描**：「任务」页面提交一个 `元数据解析` 任务，UniRhy 会扫描所有未识别的音频文件。
4. **可选：转码**：再提交一个 `转码` 任务，把无损音频转出 Opus / MP3 供低带宽端使用。
5. **创建其他账户**：「设置 → 账号管理」为家庭成员或团队添加普通账户。
6. **客户端登录**：用桌面或移动客户端登录同一账户，开始多端同步播放。

详细的功能说明见 [音乐库与存储节点](/zh/docs/usage/library) 与 [多设备播放同步](/zh/docs/usage/playback-sync)。
