server 启动后访问 `http://<服务器地址>:8654`（端口与 compose 中的映射一致）。前端检测到数据库尚未初始化时，会自动跳转到 `/init`，进入两步初始化向导。

## 第一步：创建管理员账户

第一步创建实例唯一的管理员账户。该账户拥有实例的完全控制权——后续创建其他账户、配置存储节点、安装插件等均通过该账户进行。

| 字段   | 说明                                                   |
| ------ | ------------------------------------------------------ |
| 账号名 | 登录用户名，建议简短小写英文                           |
| 邮箱   | 用于找回密码或后续通知（当前版本仅作标识，无邮件功能） |
| 密码   | 至少 8 位，建议混合大小写、数字与符号                  |

> 当前版本不支持通过 UI 重置管理员密码，请妥善保存。

## 第二步：系统存储节点

第二步指定 UniRhy 用来存放扫描索引、缓存、转码产物的根路径。

> 为避免您原有的音乐库文件夹杂乱，建议将此 系统存储节点 配置为 **空文件夹** ，并在后续系统设置页面添加您的音乐库文件夹。

- **Docker 部署**：必须填挂载卷在容器内的绝对路径。按推荐的 [Docker 部署](/zh/docs/install/docker) 配置，`compose.yml` 已将宿主机 `./data` 挂载到容器内 `/data`，此处直接填 `/data` 即可。其余非挂载路径会随容器销毁而删除。
- **裸机部署**：填一个 server 用户可读写的绝对路径，例如 `/var/lib/unirhy/data`。

该根路径会被注册为 UniRhy 的**默认本地存储节点**。后续可在「设置 → 存储节点」中追加更多本地节点或 OSS（S3 兼容）节点；UniRhy 支持多存储节点并行扫描。

## 完成初始化

点击「初始化」后自动跳转至登录页（`/login`），使用刚创建的管理员账户登录即可进入主界面。

## 接下来

进入主界面后建议按以下顺序配置：

1. **存储节点**：在「设置 → 存储节点」检查或补充节点配置。
2. **配置音乐文件夹**：将您已有的音乐文件夹配置为一个新的只读存储节点。目录结构无强制要求，UniRhy 按音频内嵌元数据组织。
3. **触发扫描**：在「任务」页面提交 `元数据解析` 任务，扫描所有未识别的音频文件。
4. **可选：转码**：提交 `转码` 任务，把无损音频转出 Opus / MP3 供低带宽端使用。
5. **创建其他账户**：在「设置 → 账号管理」为家庭成员或团队添加普通账户。
